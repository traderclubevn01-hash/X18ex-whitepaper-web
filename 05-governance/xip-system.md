# XIP — X18 Improvement Proposals

## Improvement Proposal System

### XIP Process

```
1. Draft → 2. Discussion (7 days) → 3. Voting (5 days) → 4. Execution
```

### XIP Types

| Type | Example | Quorum |
|:---|:---|:---:|
| **XIP-T (Technical)** | Protocol upgrade, new chain support | 10% supply |
| **XIP-E (Economic)** | Fee changes, burn rate adjustment | 15% supply |
| **XIP-G (Growth)** | Grant proposals, partnerships | 5% supply |
| **XIP-X (Emergency)** | Security fixes, emergency pause | 3/5 council + 5% supply |

### Thresholds
- **Proposal Submission**: Stake 100 $X18 (refunded if proposal passes)
- **Approval**: >50% of votes cast (simple majority)
- **Execution Timelock**: 48h for standard, 24h for emergency
